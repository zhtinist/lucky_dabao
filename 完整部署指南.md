# 加密网站完整部署指南

## 🚀 一键部署步骤

### 第一步：运行部署脚本
双击运行 `deploy.bat` 文件，按照提示操作。

### 第二步：创建GitHub仓库
1. 访问 [GitHub](https://github.com)
2. 点击右上角的 "+" 号，选择 "New repository"
3. 仓库名称填写：`lucky_dabao`
4. 设置为 **Private**（私有仓库）
5. 不要勾选任何初始化选项
6. 点击 "Create repository"

### 第三步：连接本地仓库到GitHub
复制GitHub给出的仓库地址，然后在命令行中运行：

```bash
# 添加远程仓库（替换为你的实际地址）
git remote add origin https://github.com/zhtinist/lucky_dabao.git

# 推送source分支
git push -u origin source
```

### 第四步：设置密码Secret
1. 在GitHub仓库页面，点击 "Settings" 标签
2. 在左侧菜单中找到 "Secrets and variables" > "Actions"
3. 点击 "New repository secret"
4. Name 填写：`SITE_PASSWORD`
5. Secret 填写：`LuckyDabao2026`
6. 点击 "Add secret"

### 第五步：创建main分支
```bash
# 创建并切换到main分支
git checkout --orphan main

# 清空工作目录
git rm -rf .

# 创建README
echo "# 加密文件存放分支" > README.md

# 提交并推送
git add README.md
git commit -m "初始化main分支"
git push -u origin main

# 切换回source分支
git checkout source
```

### 第六步：启用GitHub Pages
1. 在GitHub仓库页面，点击 "Settings" 标签
2. 滚动到 "Pages" 部分
3. Source 选择：**Deploy from a branch**
4. Branch 选择：**main**
5. Folder 选择：**/ (root)**
6. 点击 "Save"

## 📁 日常使用

### 添加新文件
1. 将要加密的文件放入 `src` 目录
2. 双击运行 `quick-update.bat`
3. 输入提交信息（可选）
4. 等待自动部署完成（约2-3分钟）

### 手动更新
```bash
git add .
git commit -m "添加新文件"
git push origin source
```

## 🔧 高级设置

### 修改密码
1. 在GitHub仓库的 Settings > Secrets and variables > Actions
2. 找到 `SITE_PASSWORD`，点击编辑
3. 输入新密码并保存
4. 推送任意更改触发重新部署

### 自定义域名
1. 在GitHub仓库的 Settings > Pages
2. 在 "Custom domain" 中输入你的域名
3. 配置DNS记录指向GitHub Pages

## 🛠️ 故障排除

### GitHub Actions失败
1. 检查 Actions 标签页的错误日志
2. 确认 `SITE_PASSWORD` Secret已正确设置
3. 确保 `src` 目录中有文件

### 网站无法访问
1. 等待5-10分钟让部署完成
2. 检查GitHub Pages设置是否正确
3. 确认main分支有内容

### 密码验证失败
1. 确认输入密码正确
2. 检查浏览器控制台是否有错误
3. 重新推送触发部署
