# 加密网站使用说明

## 首次部署

1. **运行部署脚本**
   ```
   双击运行 deploy.bat
   ```

2. **按照脚本提示完成GitHub设置**
   - 创建GitHub仓库
   - 设置密码Secret
   - 启用GitHub Pages

## 日常使用

### 添加新文件
1. 将要加密的文件放入 `src` 目录
2. 双击运行 `quick-update.bat`
3. 输入提交信息（可选）
4. 等待自动部署完成

### 手动操作
```bash
# 添加文件
git add .

# 提交更改
git commit -m "添加新文件"

# 推送到GitHub
git push origin source
```


## 工作原理

1. **文件加密**: GitHub Actions使用AES算法加密`src`目录中的所有文件
2. **网页生成**: 自动生成带密码验证的访问页面
3. **自动部署**: 加密后的文件和网页部署到GitHub Pages

## 注意事项

- 确保文件放在 `src` 目录中
- 每次推送到source分支都会触发重新加密
- 密码存储在GitHub Secrets中，安全可靠
- 支持各种文件格式（文本、HTML、图片等）

## 故障排除

### 如果GitHub Actions失败
1. 检查GitHub Secrets中是否正确设置了 `SITE_PASSWORD`
2. 确保仓库启用了GitHub Actions
3. 查看Actions日志了解具体错误

### 如果网站无法访问
1. 确认GitHub Pages已启用
2. 检查main分支是否有内容
3. 等待几分钟让部署完成

### 如果密码验证失败
1. 确认输入的密码正确
2. 检查GitHub Secrets中的密码设置
3. 重新推送触发部署
